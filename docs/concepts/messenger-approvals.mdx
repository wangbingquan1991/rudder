Messenger is Rudder's communication and attention surface. It brings together chat, issue threads, approval requests, failed runs, budget alerts, and join requests.

![Messenger approval queue](../images/messenger-approvals.png)

Messenger is not meant to replace the issue system. It helps route and clarify work, then keeps attention attached to durable objects.

## Use the right surface

| Use case | Surface |
| --- | --- |
| Clarify a request or attach context | Messenger chat |
| Decide whether an operation may proceed | Approval |
| Execute long-running work | Issue |
| Explain progress or close-out | Issue comment |
| Recover attention after failure or blocker | Messenger attention stream |

## Chat intake

Use chat when you need to explain a request, attach context, ask for routing, or turn a conversation into a proposed issue.

Chat can link context such as issues, projects, and agents. Depending on organization settings, chat-created issues may require manual approval or be created automatically.

## Approvals

Approvals are structured decision objects. Rudder uses approvals for operations such as:

- hiring an agent
- approving strategy
- overriding a budget stop
- creating issues from chat
- approving a chat-requested operation

Approvals can be pending, revision requested, approved, rejected, or cancelled.

## Attention streams

Messenger also surfaces system threads for failed runs, budget alerts, and join requests. These threads keep operational attention visible without forcing users to inspect every page individually.

Follow-up threads are attention prompts, not replacement tasks. Use them when an issue missed close-out, a review decision needs assignee action, a blocker needs a human answer, or a comment asks the owner to resume work. The next step should still be recorded on the issue so close-out stays inspectable.

## When to use what

Use chat to clarify. Use approvals to decide. Use issues to execute. Use comments and documents to preserve the work record.

If a conversation becomes important enough to assign, spend budget on, review,
or revisit later, convert it into an issue.
