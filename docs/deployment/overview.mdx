Rudder supports local-first operation and authenticated deployments.

![Instance settings](../images/instance-settings.png)

## Runtime modes

| Mode | Exposure | Human auth | Primary use |
| --- | --- | --- | --- |
| `local_trusted` | n/a | No login required | Single-operator local machine workflow |
| `authenticated` | `private` | Login required | Private-network access such as Tailscale, VPN, or LAN |
| `authenticated` | `public` | Login required | Internet-facing or cloud-hosted deployment |

## Local trusted

`local_trusted` is optimized for a fast local setup. It binds to loopback and avoids login friction.

## Authenticated private

Authenticated private deployments require login while allowing lower-friction private-network URL handling.

## Authenticated public

Authenticated public deployments require login and stricter public URL checks. Use this mode for internet-facing installations.

## Documentation site deployment

This documentation site is a Mintlify project under `docs/`.

Local development:

```bash
pnpm docs:dev
```

Validation:

```bash
pnpm docs:validate
```

Deployment should be wired through Mintlify's GitHub integration for the `docs/` subdirectory, then pointed at `doc.rudder.zeeland.studio` as the production custom domain.

<Warning>
  This repository does not currently contain Vercel or Mintlify project metadata for `doc.rudder.zeeland.studio`. Domain binding requires access to the Mintlify workspace and DNS provider.
</Warning>
