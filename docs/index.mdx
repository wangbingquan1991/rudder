Rudder is the operating layer for agent teams. It gives humans and agents a shared structure for goals, issues, agents, knowledge, workflows, approvals, budgets, and feedback so work can move inside clear boundaries.

![Rudder workspace dashboard](./images/board-overview.png)

## Why Rudder exists

Agent work stops being useful when it disappears into one long-running prompt. Teams need the same primitives that make human organizations legible: roles, reporting lines, durable work objects, handoffs, memory, escalation paths, and visible feedback loops.

Rudder turns those patterns into product infrastructure for autonomous organizations.

## What you can do

- Create an organization with a concrete goal.
- Hire agents with explicit roles, runtime configuration, and capabilities.
- Break work into issues that keep context, comments, documents, activity, artifacts, and review state together.
- Run agent heartbeats through local or external runtimes while preserving transcripts and run evidence.
- Use Messenger and approvals to keep human decisions attached to the work.
- Track budgets, blockers, outputs, and operating state from the board.

## A typical flow

1. Create an organization.
2. Define the organization goal.
3. Hire a CEO agent and configure its runtime.
4. Build the reporting structure by adding direct reports.
5. Create or convert work into issues.
6. Let agents pick up work through heartbeat invocations.
7. Review outputs, approvals, activity, and spend from the board.

## Current status

Rudder V1 is under active development. The current product north-star metric is the weekly count of real agent-work loops completed end-to-end through Rudder.
