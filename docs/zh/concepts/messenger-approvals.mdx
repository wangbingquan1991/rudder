Messenger 是 Rudder 的沟通和注意力界面。它把聊天、任务线程、审批请求、失败运行、预算提醒和加入请求放在一起。

Messenger 不用于替代任务系统。它帮助路由和澄清工作，然后把注意力保留在持久对象上。

![Messenger 和审批](../../images/messenger-approvals.png)

## 聊天入口

当你需要解释请求、附加上下文、请求路由，或把对话变成任务提案时，使用聊天。

聊天可以关联任务、项目和 agent 等上下文。根据组织设置，从聊天创建的任务可能需要人工审批，也可能自动创建。

## 审批

审批是结构化决策对象。Rudder 用审批处理以下操作：

- 雇佣 agent
- 批准策略
- 覆盖预算停止条件
- 从聊天创建任务
- 批准聊天请求的操作

审批可以处于 pending、revision requested、approved、rejected 或 cancelled 状态。

## 注意力流

Messenger 也会展示失败运行、预算提醒和加入请求等系统线程。这些线程让运营注意力保持可见，而不需要用户逐页检查。

Follow-up 线程是注意力提示，不是替代任务。任务缺少 close-out、评审决定需要 assignee 行动、阻塞需要人类回答，或评论要求负责人恢复工作时，使用 follow-up。下一步仍应记录在任务上，确保 close-out 可检查。

## 什么时候用什么

用聊天澄清。用审批决策。用任务执行。用评论和文档保留工作记录。
