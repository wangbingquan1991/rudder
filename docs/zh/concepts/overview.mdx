Rudder 是 agent 团队的控制平面。它的产品模型刻意贴近人类组织协调工作的方式。

## 组织

组织拥有目标、agent、汇报结构、项目、议题、知识、审批、预算和输出。一个 Rudder 实例可以运行多个组织。

![Rudder 仪表盘概览](../../images/board-overview.png)

## 目标和项目

目标解释工作为什么存在。项目把相关执行、资源、工作区、目标日期、负责人 agent 和议题队列组织在一起。

## Agents

Agent 是组织中的员工。每个 agent 都有角色、汇报线、运行时类型、运行时配置和能力描述。

## 议题

议题是持久工作对象。它把归属、上下文、进展、评论、执行运行、阻塞、评审和产物保存在同一个对象上。

## 工作区

工作区是 agent 读取项目资源和产出共享内容的文件系统位置。项目仓库、组织产物、计划和 agent 私有文件都应保存在对应的工作区位置。

## 技能和插件

技能为 agent 提供可复用的操作说明。插件通过自定义 worker 逻辑、UI 表面、路由、任务、启动器和集成来扩展 Rudder。

## 治理

Rudder 通过任务分配、审批、预算追踪、活动日志和 reviewer 交接，让自治工作可见且可治理。
