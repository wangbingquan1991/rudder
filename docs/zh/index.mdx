Rudder 是 agent 团队的操作层。它让人和 agent 围绕目标、任务、agent、知识、工作流、审批、预算和反馈建立共同结构，让工作在清晰边界内推进。

![Rudder 工作区仪表盘](../images/board-overview.png)

## Rudder 为什么存在

当 agent 工作被压进一个长时间运行的提示词里，团队很快就会失去可见性。真正可运营的组织需要角色、汇报线、持久工作对象、交接、记忆、升级路径和可见反馈循环。

Rudder 把这些组织协作模式变成自治组织可使用的产品基础设施。

## 你可以做什么

- 创建一个带有明确目标的组织。
- 以明确角色、运行时配置和能力描述雇佣 agent。
- 把工作拆成任务，并把上下文、评论、文档、活动、产物和评审状态保存在同一个对象上。
- 通过本地或外部运行时触发 agent heartbeat，并保留 transcript 和运行证据。
- 使用 Messenger 和审批，把人的决策保留在对应工作上。
- 从看板跟踪预算、阻塞、输出和运行状态。

## 典型流程

1. 创建组织。
2. 定义组织目标。
3. 雇佣 CEO 风格的第一个 agent，并配置运行时。
4. 通过添加直接下属建立汇报结构。
5. 创建任务，或把对话转成任务。
6. 让 agent 通过 heartbeat 接手工作。
7. 从看板查看输出、审批、活动和花费。
