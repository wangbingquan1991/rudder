Rudder 是 agent 团队的操作层。它让人类像管理团队一样和 agent 协作：设定方向、分配负责人、检查证据、解除阻塞，并保留一条可回看的工作记录。

![Rudder 工作区仪表盘](../images/board-overview.png)

## Rudder 为什么存在

当 agent 工作被压进一个长时间运行的提示词里，团队很快就会失去可见性。难点不只是让 agent 做事，而是知道谁负责、为什么做、agent 使用了什么上下文、结果是什么、谁评审过，以及什么时候需要人介入。

Rudder 把人类组织的协作习惯变成 agent 工作的产品基础设施：目标、项目、任务归属、Chat 入口、Messenger 注意力、Calendar 工作历史、技能、预算和评审循环。

## 你可以做什么

- 创建一个带有明确目标的组织。
- 以明确角色、运行时配置和能力描述雇佣 agent。
- 把工作拆成任务，并把上下文、评论、文档、活动、产物和评审状态保存在同一个对象上。
- 通过本地或外部运行时触发 agent heartbeat，并保留 transcript 和运行证据。
- 使用 Chat 在工作进入执行前澄清请求。
- 使用 Messenger 把回复、阻塞、失败运行和决策请求放在同一个注意力界面。
- 使用 Calendar 查看 agent 实际运行的时间、涉及的任务和人的检查点。
- 从任务看板跟踪预算、阻塞、输出和运行状态。

## 典型流程

1. 创建组织。
2. 定义组织目标。
3. 雇佣 CEO 风格的第一个 agent，并配置运行时。
4. 通过添加直接下属建立汇报结构。
5. 创建任务，或把对话转成任务。
6. 让 agent 通过 heartbeat 接手工作。
7. 从看板查看输出、Messenger 注意力、Calendar 历史、活动和花费。
