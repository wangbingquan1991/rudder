# Rudder v0.2.0

Released: 2026-05-10

Rudder 0.2.0 focuses on reviewer-aware issue workflows, richer Messenger coordination, release/desktop distribution hardening, and better run and cost visibility for agent teams.

## Highlights

- Added reviewer-aware issue workflows, including reviewer display on issue board cards, blocked reviewer handoffs, reviewer closeout recovery, and clearer approval payload rendering.
- Improved Messenger-driven issue work with selected-agent runtime routing, plan-mode approval gates, persistent streaming across navigation, hidden-conversation unread handling, Markdown rendering, mentions, image attachments, and draft issue autosave isolation.
- Expanded agent and operator visibility with organization cost trends, per-agent run cost charts, cost breakdown avatars, transcript/tool rendering improvements, and Langfuse prompt I/O exposure.
- Hardened local runtime and CLI surfaces with git identity guards/settings, managed runtime auth preservation, server-backed issue search, CLI runtime caching, issue comment image upload, and safer issue/document context in agent wakes.
- Improved Desktop release and update behavior with update-channel settings, preview/recovery surfaces, deferred active-run updates, and public packaging/CI install fixes.
- Refined product polish across chat, issue cards, tabs, tooltips, priority visuals, onboarding seeds, and remaining localization cleanup.

## Install

```sh
npx @rudderhq/cli@latest start
```

After the persistent CLI is installed, `rudder start` uses the same CLI surface when it resolves to the same version.

## Notes

This release promotes the `0.2.0` package line from public canaries to the stable npm `latest` dist-tag. Desktop assets are attached to the `v0.2.0` GitHub Release and verified by checksum before installation.
